# 2026-07-10 工作日志

## v3.11.6 — Create RMA Case 页 RMA Items 修复 + 日期图标 + Basic 卡片收紧

背景：用户反馈 v3.11.5 的 RMA Items 明细区"被改坏"——字段被挤出/不可见（只到 Return Reason），Return Reason 显示 `Functio...`，日期图标跑到输入框外很远。

### 根因
1. RMA Items 表格用 `width:max-content; table-layout:fixed`——fixed 布局下 max-content 在部分场景不可靠，容器窄于列宽合计时仍被压缩，字段截断/不可见。
2. 日期图标：`.date-picker-wrap` 缺少 `position:relative`，内部绝对定位的日历按钮锚定到最近的定位祖先 = 全屏浮层 `.record-fs-overlay`（position:fixed），因此按钮被推到最右侧边缘。

### 修复（均不改业务/审批/API/DB/权限红线）
- 表格改为 `width:100%; min-width:1760px; table-layout:fixed`（用户明确要求的 min-width 方案）：列宽保底 1760px，容器更窄时由 `.detail-items-table-wrapper{overflow-x:auto}` 横向滚动，绝不压缩。
- 表头列宽按新规格：Channel120 / Shop200 / SKU240 / SN160 / Order180 / Qty80 / Return Reason220 / Issue Description320 / Attachments150 / Actions80（合计 1750px）。
- Actions 列 td 加 `padding-right:16px`，删除按钮不贴边；`position:sticky;right:0` 保持滚动时始终可见、不被挤出。
- Return Reason 列 220px + 输入框 `width:100%`，"Functional Issue" 完整显示（标准英文已在 v3.11.5 落地）。
- 日期图标：补 `.date-picker-wrap{position:relative;display:block;width:100%}`（合并了原 display:flex 的旧规则，去掉重复）；Basic 卡片内 `.form-card--basic .date-picker-wrap{display:inline-block;width:240px}` 约束包裹层宽度，使图标落在输入框内部右侧。
- Basic Information 卡片收紧：`.form-card--basic` 头部 padding 12→10 18px、内容 14→12 18px，减少垂直留白；RMA Date 输入框宽约 240px（不横跨整行）。
- 版本三处同步（title / sidebar-footer / package.json → 3.11.6）。

### 验证
- 三 locale + package.json JSON 合法；node --check 内联脚本语法 OK。
- Render 部署后线上 curl 确认：v3.11.6、`min-width:1760px`、列宽（sku240/reason220/actions80）、`date-picker-wrap{position:relative...}` 均生效。
