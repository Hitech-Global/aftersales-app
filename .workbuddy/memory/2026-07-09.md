## 线上修复：进入系统后概览页空白 (v3.9.15 / v3.9.16)

**现象**: 登录进入后默认"售后概览"侧栏选中、标题正确，但主内容区域完全空白；Network 接口有返回。

**根因（两个叠加）**:
1. `enterApp()` 只调用 `handleHashRoute()`；用户 URL 是 `?lang=en-US` query 参数（**无 hash**），`handleHashRoute` 在 `if(!hash) return` 直接退出 → 没有任何 `showPage` 被调用 → `page-overview` 从未激活、概览从未渲染。

**修复 (index.html)**:
- `enterApp()` 末尾加：`if(!document.querySelector('.page.active')){ showPage('overview'); }` → 无 hash 时默认进入概览
- 重写 `renderOverview`：**数据/卡片/最近记录渲染 与 图表渲染彻底解耦**。主内容用独立 try 渲染（异常只显示空状态，不再清空）；图表用新增 `safeRenderChart()` 包裹（`typeof Chart==='undefined'` 直接跳过 + 各自 try/catch），图表失败绝不影响主内容
- 新增 `ovCard`/`ovCardClick`/`safeRenderChart` 辅助函数
- 概览看板新增兜底内容：今日售后数 / 待审批 / 待处理 / 已完成（卡片）+ 最近售后记录列表（`ov-recent` 容器，空数据显示"暂无数据"）
- 补充 i18n key：`overview.card.today/completed`、`overview.recent.title/empty/pending`（en-US/zh-CN/id-ID）

**验证方式（无浏览器也能验）**: Node + vm 沙箱加载内联脚本并真实执行 `renderOverview`（桩 DOM + fetch 返回样例 + Chart 故意不定义）→ 确认不抛错、TEST 标记/卡片/空状态均渲染，且 Chart 缺失时主内容未被清空。

**部署**:
